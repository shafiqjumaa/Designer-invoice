# DesignerFlow — نظام إدارة أعمال المصمم

تطبيق ويب واحد (ملف `index.html` واحد فقط) لإدارة أعمال مصمم فريلانس: العملاء، العقود، الفواتير، المصاريف، الاشتراكات، قائمة الانتظار، والأهداف.

## رفعه على GitHub Pages
1. أنشئ مستودع جديد باسم مناسب، مثلاً: `designerflow`.
2. ضع ملف `index.html` في جذر المستودع.
3. من إعدادات المستودع (Settings → Pages) فعّل GitHub Pages على فرع `main`.
4. بعد دقيقة سيصبح الرابط: `https://username.github.io/designerflow/`

## ⚠️ مهم قبل الرفع
كلمة سر لوحة التحكم مكتوبة بشكل صريح داخل الكود (`PASSWORD = "Reem"`). أي شخص يفتح "View Source" على GitHub Pages يمكنه قراءتها، لأن هذا تطبيق front-end بالكامل بدون خادم. لذلك:
- **غيّر كلمة السر** قبل الرفع (ابحث عن `const PASSWORD` في الكود).
- إذا أردت حماية أقوى، اجعل المستودع **Private** (لكن GitHub Pages على الحسابات المجانية يتطلب أحياناً Public — تحقق من خطتك)، أو استخدم استضافة بكلمة سر حقيقية بدل هذا الأسلوب.

## الروابط الجديدة للعملاء
كل البيانات تُخزَّن في متصفحك فقط (`localStorage`) — لا يوجد خادم. لذلك الروابط العامة صُممت بحيث:
- **رابط الاستبيان**: `?client=1` — يفتح استمارة "طلب مشروع جديد" مباشرة بدون المرور بشاشة تسجيل الدخول. عند الإرسال يتم تحويل العميل تلقائياً إلى واتساب (الرقم الذي تضبطه في الإعدادات) برسالة جاهزة فيها كل إجاباته.
- **رابط الفاتورة**: يُنسخ من صفحة "الفواتير" (أيقونة 🔗 بجانب كل فاتورة). الرابط يحمل بيانات **هذه الفاتورة فقط** مُرمّزة داخل الرابط نفسه — لا يكشف أي بيانات أخرى (عملاء، فواتير أخرى، أرقام مالية عامة...).

كل هذه الروابط تُنسخ جاهزة من صفحة **الإعدادات**.

## الإعدادات الجديدة
- **رابط موقعك الخاص**: يظهر للعميل في صفحة الفاتورة العامة.
- **رقم واتساب لاستلام الطلبات**: يستقبل ردود استبيان العملاء تلقائياً.

## ملاحظة تقنية
تم تفعيل الحفظ الحقيقي في `localStorage` (سابقاً كانت البيانات تُحفظ في الذاكرة فقط وتُفقد عند إغلاق الصفحة).
